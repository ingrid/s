Basically we are placing the frames side by side, with no padding between them. If you do not want something drawn in game, then it should not be drawn on the sprite sheet, so there are no borders and no backgrounds on the sprite (for the purposes of this webpage a white background is displayed, but you want to make the background transparent in practice). Each frame starts where the old frame ended. Notice that we only need to include the unique frames, the first frame can be reused, so in the actual animation we play the first frame, the second frame, the first frame, the third frame, and repeat. Final Notes

There are variations on sprite sheet formats you can read about later, but this is the most simple and most common implementationt, and it will work pretty much everywhere. If you find you need to do things differently, you can research how to do so when the problem arises, otherwise, for the sake of getting started here are some basic guidelines to start off with (again, just for the sake of getting started, you can do things differently later if you understand the sprite engine of your specific engine).

  • Each sprite sheet should be self contained, meaning it should be in its own image file. Don't put 2 sprite sheets in a single image file, and don't split the sprite sheet for one character into 2 files.
  • All the frames in a give sprite sheet should be the same size. Different sprite sheets can have different size frames, but sprite sheets much be consisten with themslves.
  • There should be no 'padding' between frames. Frames are laid side by side with no spacing.
  • Sprites with direction should always face left. The sprite engine can mirror the image for you when you need it to face to the right.
  • You probably want a transparent background for most character sprites you are making.
  • Don't draw a border on a frame unless you want that border to show up in game.
  • Remeber alphas are fine in sprite sheets :D
